## Directory Purposes

**docs/:**
- Purpose: GitHub Pages deployment directory (static HTML output)
- Contains: Compiled HTML pages, CSS, JavaScript, images, and Quarto site libraries
- Key files:
  - `index.html`: Rendered homepage with particles animation
  - `blog.html`: Blog listing page
  - `posts/*/index.html`: Individual blog post pages
  - `styles.css`: Compiled CSS (copy from source)
  - `site_libs/`: Bootstrap, Quarto navigation, OJS runtime, syntax highlighting

**_freeze/:**
- Purpose: Cache for executed R/Python code and outputs during build
- Contains: Serialized execution results to avoid re-running expensive computations
- Key files: `.json` files with cached outputs, `.png` figures from plots
- Generated: Yes (git-ignored in production)
- Committed: No (in .gitignore)

**.quarto/:**
- Purpose: Quarto internal build state and indices
- Contains: Cross-reference metadata, preview lock file, search indices
- Generated: Yes
- Committed: No

**posts/:**
- Purpose: Blog post content directory
- Contains: Subdirectories named by date (YYYY-MM-DD format), each with a `.qmd` file
- Structure: `posts/YYYY-MM-DD/YYYY-MM-DD.qmd` (e.g., `posts/2025-11-04/2025-11-04.qmd`)
- Key files: Individual `.qmd` files with markdown + code cells
- Example: `posts/2025-11-04/2025-11-04.qmd` contains TidyTuesday lead analysis with R code

**images/:**
- Purpose: Source images used in content
- Contains: JPG and PNG files for hero headshot and blog post images
- Key files:
  - `headshot.jpg`: Profile photo (150x150px, circular crop at face)
  - `flywheel.jpg`: Hackathon result/team photo
- Generated: No
- Committed: Yes

**.vscode/:**
- Purpose: VS Code workspace configuration
- Contains: `settings.json` (editor preferences) and `extensions.json` (recommended extensions)
- Generated: No
- Committed: Yes

## Naming Conventions

**Files:**
- Quarto documents: lowercase with hyphens (`index.qmd`, `blog.qmd`, `projects.qmd`)
- SCSS/CSS: lowercase with hyphens (`custom.scss`, `styles.css`)
- HTML templates: lowercase with hyphens (`title-block.html`)
- JSON config: kebab-case (`particles.json`)
- Blog posts: date-based directory structure (`posts/YYYY-MM-DD/YYYY-MM-DD.qmd`)

**Directories:**
- Internal Quarto: underscore prefix (`.quarto`, `_freeze`, `_quarto.yml`)
- GitHub: dot prefix (`.github`, `.git`, `.vscode`)
- Content subdirectories: lowercase plural (`posts`, `images`, `docs`)
- Generated output: all lowercase (`docs`)

**CSS Classes:**
- Page-specific body classes: `page-{section}` (e.g., `page-blog`, `page-work`, `page-volunteer`)
- Component classes: lowercase with hyphens (`.hero-section`, `.hero-content`, `.btn-primary`, `.card`)
- Bootstrap grid: `.grid`, `.g-col-12`, `.g-col-md-4`
- SCSS variables: kebab-case for colors (`$phthalo-deep`, `$oxblood-light`, `$tyrian`)

**Variables (SCSS/CSS Custom Properties):**
- Color palettes: semantic names (`$primary`, `$link-color`, `$code-block-bg`)
- Section palettes: descriptive names (`$oxblood`, `$oxblood-light`, `$tyrian`, `$tyrian-light`, `$amber`, `$amber-light`)
- Size/spacing: numeric or descriptive (`$light-teal-bg`, `$near-white`, `$code-bg`)

## Where to Add New Code

**New Blog Post:**
- Create directory: `posts/YYYY-MM-DD/`
- Create file: `posts/YYYY-MM-DD/YYYY-MM-DD.qmd`
- Include frontmatter:
  ```yaml
  ---
  title: "Post Title"
  author: "Sean Thimons"
  date: "YYYY-MM-DD"
  description: "Short summary for listing"
  categories: [tag1, tag2, tag3]
  ---
  ```
- Content renders with `page-blog` body class automatically (color scheme: Oxblood)
- Figures/outputs: Store in `posts/YYYY-MM-DD/output/` (added to .gitignore if auto-generated)

**New Page (e.g., Research Notes, Resources):**
- Create file: `newpage.qmd` in project root
- Add to navbar in `_quarto.yml` under `website.navbar.left`
- Optional: Add `format.html.body-classes: page-newpage` for custom color scheme
- Define color scheme in `custom.scss` if needed (add new section after line 343)

**New Section-Specific Color Scheme:**
- Add color palette variables at top of `custom.scss` (after line 18):
  ```scss
  $newsection:      #HEXCODE;
  $newsection-light: #HEXCODE;
  ```
- Add section rules block after line 343:
  ```scss
  body.page-newsection {
    // Links
    #quarto-content a,
    .quarto-title a,
    main a {
      color: $newsection !important;
      text-decoration-color: rgba($newsection, 0.3);
      &:hover {
        text-decoration-color: $newsection;
        color: darken($newsection, 10%) !important;
      }
    }
    // ... add more selectors as needed
  }
  ```
- Add `body-classes: page-newsection` to `.qmd` file frontmatter

**Utility Scripts or R Functions:**
- Create in `posts/YYYY-MM-DD/YYYY-MM-DD.qmd` as code cells if post-specific
- Create `.R` file in project root if reusable (currently no utility files detected; would go in `R/` if created)

**Images:**
- Store in `images/` directory with lowercase kebab-case names
- Reference in markdown as `![alt text](images/filename.jpg)`
- Consider size/format: headshots as JPG (circular with `border-radius`), diagrams as PNG

## Special Directories

**docs/ (Output):**
- Purpose: GitHub Pages deployment directory (served at `seanthimons.github.io`)
- Generated: Yes (by Quarto build)
- Committed: Yes (required for GitHub Pages)
- Build command: `quarto render` (reads from source, writes to `docs/`)
- Never edit directly; always regenerate via `quarto render`

**_freeze/ (Cache):**
- Purpose: Cache executed code results to speed up rebuilds
- Generated: Yes (by Quarto)
- Committed: No (in .gitignore)
- When to clear: If R/Python environment changes or code execution needs refresh
- Clear command: `rm -r _freeze/` then re-run `quarto render`

**.quarto/ (Quarto State):**
- Purpose: Quarto internal state and indices
- Generated: Yes
- Committed: No (in .gitignore)
- User shouldn't touch this directory

## Configuration File Purposes

**_quarto.yml:**
- Defines project type: `type: website`
- Sets output directory: `output-dir: docs`
- Configures navbar (left/right links)
- Sets theme: `[flatly, custom.scss]` (Bootstrap 5 flatly + custom overrides)
- CSS files: `styles.css`
- Execution: `freeze: true` (cache results locally; never re-execute on CI)

**seanthimons.github.io.Rproj:**
- RStudio project marker
- Stores RStudio-specific settings (workspace, history)
- Not required for Quarto builds; convenience for R IDE users

**air.toml:**
- Live reload development server configuration
- Monitors source files for changes
- Auto-rebuilds and refreshes browser
- Used during development: `air` command

**.nojekyll:**
- Tells GitHub Pages to skip Jekyll processing
- Necessary because Quarto generates all HTML upfront; Jekyll would interfere
- Just an empty marker file
